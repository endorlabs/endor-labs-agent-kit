# Catalog signing keys

This directory holds the **public** key used to verify the signed `catalog.json`
release artifact. No private key material lives here or anywhere in the repo --
signing is HSM-backed via Azure Key Vault over OIDC-federated CI (see
`RELEASES.md`).

## `catalog-signing-public-key.pem` (added under AI-418)

The ECDSA-P256 (ES256) **public** key exported from the Azure Key Vault
catalog-signing key. It is intentionally **not** committed yet:

- Committing it is a security-critical, CODEOWNERS-reviewed action performed when
  the Key Vault signing key is provisioned (AI-418), not generated by tooling.
- Until it exists, the release workflow's signature step stays gated
  (`ENDOR_ARTIFACT_SIGNING_ENABLED`), and `endor-agent-kit verify-catalog-signature`
  fails closed if pointed at a missing or invalid key -- so signing can never be
  silently enabled against a stand-in key.

The same public key is shipped to apiserver (AI-351) for pinning. Rotation is
documented in `RELEASES.md`.
